### Добавление коммита
Добавляем в индекс все изменения в текущей директории и поддиректориях:
1. Новые файлы (untracked).
2. Измененные файлы (modified).
3. Удаленные файлы (deleted).

git add . — самый популярный и короткий вариант. Удобно, если вы находитесь в корне проекта. Если вы находитесь в подпапке, она добавит изменения только из этой папки.

```bash
git add .
```

git add -A — гарантированно добавит изменения во всем рабочем дереве, даже если вы запустите её из глубокой подпапки.

```bash
git add -A
```

Важный нюанс:
Обе команды игнорируют файлы, прописанные в .gitignore. Чтобы принудительно добавить игнорируемый файл, нужно использовать флаг -f (force).


Добавить описание к коммиту.

```bash
git commit -m "Initial commit"
```

Команда git push — это финальный этап публикации кода. Она берет все ваши локальные коммиты и отправляет (загружает) их в удаленный репозиторий.

```bash
git push
```

git push -u origin <branch> — используется при первой отправке новой ветки. Флаг -u связывает локальную ветку с удаленной, чтобы в следующий раз можно было писать просто git push.

Команда git pull — это способ «забрать» свежие изменения из удаленного репозитория и сразу добавить их в свой код на компьютере.
Эта команда объединяет в себе два действия:
1. git fetch — скачивает данные из облака, которых у вас еще нет.
2. git merge — автоматически вливает эти данные в вашу текущую ветку.

Если git push — это «отдать», то git pull — это «взять».
